# Plan: Wire Up Transaction Creation

## Problem
When a user deposits, withdraws, or transfers money, the account balance changes but **no Transaction record is created**. The `Transaction` entity, controller, and query handlers all exist, but `Transaction.Create()` is never called.

## Approach
Add `Transaction.Create()` calls directly in the existing command handlers (`DepositCommandHandler`, `WithdrawCommandHandler`, `TransferCommandHandler`). This is the simplest, most direct approach. We can refactor to Domain Events later.

## Files to Modify

### 1. `DepositCommandHandler` — create Credit transaction after deposit
**File:** `src/CoreBanking.Application/Accounts/Commands/Deposit/DepositCommandHandler.cs`
- Inject `ITransactionRepository`
- After `account.Deposit(request.Amount)`, create a `Transaction` with type `Credit`
- Call `_transactionRepository.AddAsync(transaction)`

### 2. `WithdrawCommandHandler` — create Debit transaction after withdrawal
**File:** `src/CoreBanking.Application/Accounts/Commands/WithdrawCommandHandler.cs`
- Inject `ITransactionRepository`
- After `account.Withdraw(request.Amount)`, create a `Transaction` with type `Debit`
- Call `_transactionRepository.AddAsync(transaction)`

### 3. `TransferCommandHandler` — create two transactions (debit + credit)
**File:** `src/CoreBanking.Application/Accounts/Commands/TransferCommandHandler.cs`
- Inject `ITransactionRepository`
- After transfer, create:
  - Debit transaction on `fromAccount` (money going out)
  - Credit transaction on `toAccount` (money coming in)
- Both linked via `RelatedAccountId`

### 4. Verify existing queries/controllers work
- `GET /api/transactions` — should now return created transactions
- `GET /api/accounts/{id}/transactions` — should return account-specific transactions

## Transaction.Create Pattern
```csharp
var transaction = Transaction.Create(
    account: account,
    type: TransactionType.Credit,  // or Debit
    amount: request.Amount,
    currency: account.Balance.Currency,
    balanceBefore: account.Balance.Amount - request.Amount,  // before the operation
    description: "Deposit",
    relatedAccountId: null  // or other account ID for transfers
);
transaction.Complete();  // mark as completed immediately
```

Note: `balanceBefore` must be captured BEFORE the operation modifies the balance.
